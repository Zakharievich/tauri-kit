# Опциональный Python LiveKit Agent (транскрипция)

Полностью **опциональный** компонент проекта. Отвечает за:
- подключение к комнате LiveKit как **скрытый участник** (без публикации
  видео/аудио, только подписка на аудиотреки остальных участников);
- распознавание речи (STT) через **faster-whisper-server** (модель `tiny`,
  OpenAI-совместимый HTTP API);
- накопление сегментов транскрипта с временными метками `[HH:MM:SS] текст`;
- опциональное саммари через **Ollama** (`qwen2.5:0.5b`) — 3–5 пунктов;
- рассылку финального документа всем участникам комнаты по DataChannel
  (топик `transcript_final`).

Отсутствие или сбой агента **не блокирует звонок** — приложение продолжает
работать без транскрипции (graceful degradation, см. `docs/PLAN.md`
HIGH RISK 4.3).

## 1. Требования к серверу

| Конфигурация                                  | RAM  | Диск     | GPU        |
|------------------------------------------------|------|----------|------------|
| Только STT (`tiny`)                             | 4 GB | +500 MB  | не нужен   |
| STT + саммари (`tiny` + `qwen2.5:0.5b`)         | 4 GB | +1 GB    | не нужен   |

> **Примечание по точности:** модель `tiny` подходит для распознавания чёткой
> речи без сильных помех. При работе в шумных условиях или с выраженными
> акцентами рекомендуется заменить модель на `small` через переменную
> `FASTER_WHISPER_MODEL` в `.env`.

## 2. Установка

1. `pip install -r requirements.txt`
2. Установить Ollama: https://ollama.com/download
3. `ollama pull qwen2.5:0.5b`
4. `uvicorn faster_whisper_server.app:app --port 8000`
   (модель `tiny` скачивается автоматически при первом запуске)

## 3. Запуск

```bash
cp .env.example .env  # отредактировать под свой сервер
python main.py dev
```

Заполните `.env` реальными значениями (`LIVEKIT_URL`, `LIVEKIT_API_KEY`,
`LIVEKIT_API_SECRET` должны совпадать с `server/.env`). Файл `.env` уже
покрыт корневым `.gitignore` — не коммитьте секреты (HIGH RISK 4.5).

(`livekit-agents` CLI поддерживает и другие подкоманды, например `start` для
продакшн-режима — см. документацию `livekit-agents`.)

## 4. Важно

Если агент **не запущен** — основное приложение работает полностью,
транскрипция и саммари просто недоступны. **Никаких ошибок у клиентов не
возникает.**

- **faster-whisper-server недоступен** — предупреждение в лог, аудиочанк
  пропускается, работа агента продолжается.
- **Ollama недоступна/таймаут** — предупреждение в лог, финальный документ
  публикуется с `summary: ""`, агент не падает.
- **Агент вообще не запущен/упал** — фронтенд не получит сообщение по
  `transcript_final` и просто останется без транскрипта, звонок продолжает
  работать в обычном режиме.

## Переменные окружения

| Переменная             | Обязательна | Назначение                                              |
|-------------------------|-------------|----------------------------------------------------------|
| `LIVEKIT_URL`            | да          | WS URL self-hosted LiveKit SFU                            |
| `LIVEKIT_API_KEY`        | да          | API key для подключения агента к комнате                  |
| `LIVEKIT_API_SECRET`     | да          | API secret                                                 |
| `ENABLE_STT`             | нет (`true`)| Включает распознавание речи. `false` → агент завершает работу с предупреждением при старте, к комнате не подключается |
| `FASTER_WHISPER_URL`     | нет         | URL OpenAI-совместимого API faster-whisper-server (default `http://localhost:8000/v1`) |
| `FASTER_WHISPER_MODEL`   | нет         | Модель whisper (default `tiny`, при плохом качестве звука — `small`) |
| `ENABLE_SUMMARY`         | нет (`true`)| Включает запрос саммари в Ollama по завершении сессии      |
| `OLLAMA_URL`             | нет         | URL Ollama (default `http://localhost:11434`)              |
| `OLLAMA_MODEL`           | нет         | Модель Ollama (default `qwen2.5:0.5b`)                     |

## Матрица поведения (см. `docs/PLAN.md` §5)

| E2EE | Transcription toggle | ENABLE_STT | ENABLE_SUMMARY | Поведение агента |
|------|-----------------------|------------|-----------------|-------------------|
| ON   | (принудительно OFF)   | —          | —               | Агент не подключается к комнате |
| OFF  | OFF                   | any        | any             | Агент не подключается к комнате |
| OFF  | ON                    | false      | any             | Агент подключён, но STT неактивен (транскрипта нет) |
| OFF  | ON                    | true       | false           | STT активен → финальный документ без саммари |
| OFF  | ON                    | true       | true            | STT активен + саммари → финальный документ с саммари |

**Важно:** агент — обычный участник комнаты и физически не может
расшифровать медиапоток при включённом E2EE. Поэтому E2EE и Transcription на
фронтенде — взаимоисключающие toggle (см. `docs/PLAN.md` HIGH RISK 4.3).

## Формат сообщения (DataChannel-контракт, зафиксировано)

Топик `transcript_final`, JSON:

```json
{
  "type": "transcript_final",
  "version": 1,
  "payload": {
    "transcript": "[00:00:05] Привет всем\n[00:00:12] Начинаем встречу",
    "summary": "- Обсудили план на неделю\n- Согласовали дедлайн релиза"
  }
}
```

- `type` — идентификатор сообщения, всегда `"transcript_final"` для
  финального документа (зарезервировано `"transcript_live"` для будущих
  live-субтитров — в этой версии агента не публикуется).
- `version` — число, сейчас `1`; нужно для обратной совместимости при
  будущих изменениях формата `payload`.
- `payload.transcript` — полный текст транскрипта, по одной строке
  `[HH:MM:SS] текст` на сегмент.
- `payload.summary` — строка с саммари (3–5 пунктов) либо `""`, если
  `ENABLE_SUMMARY=false` или Ollama была недоступна на момент завершения
  сессии.

Фронтенд (`src/hooks/useTranscription.ts`) обязан проверять `message.type`
перед обработкой и игнорировать неизвестные типы — это защищает от ошибок
при будущем расширении протокола.

## Хранение данных

Агент **не хранит** транскрипт на диске и не отправляет его ни на один
сервер — только рассылает участникам комнаты по DataChannel. Сохранение
`.txt` происходит **локально у каждого участника** через Tauri-команду
`save_transcript` (см. `docs/PLAN.md` §5, HIGH RISK 4.3).
